4. Algorithms Details

4.1. Walk

00. Procedure get_binding_sites(H, proteinA, proteinB, alpha):
01.     A <- len(proteinA) - 20
02.     B <- len(proteinB) - 20
            (here H is 2-D-landscape-array ranging from 
            H[0][0](left,lower) to H[A][B](right,upper) 
            For simplicity, we are representing H[x][y] as graphical format)
03.     peak_hits <- maximum(H)    
04.     if (peak_hits == 0):
05.         return (-1, -1) (-1, -1)
                (meaning sites could not be found, 
                because previous attempts (site-1 or site-2) have exhausted the entire landscape)
06.     (x,y) <- get_indices_of_peak_of_H
07.     up, down <- y
08.     right, left <- x
09.     while (up < B && H[x][up] > alph*peak_hits):
10.         up <- up + 1
11.     while (down > 0 and H[x][down] > alph*peak_hits):
12.         down <- down - 1
13.     while (right < A and H[right][y] > alph*peak_hits):
14.         right <- right + 1
15.     while (left > 0 and H[left][y] > alph*peak_hits):
16.         left <- left - 1
15.     return (left, right) (down, up)
  • Find out the maximum point (lets call it THE hill)
  • set a threshold (x% of height of THE hill)
  • walk in UP direction from THE hill till we reach the threshold or boundary (whichever comes first)
  • walk in DOWN direction from THE hill till we reach the threshold or boundary (whichever comes first)
  • walk in LEFT direction from THE hill till we reach the threshold or boundary (whichever comes first)
  • walk in RIGHT direction from THE hill till we reach the threshold or boundary (whichever comes first)
  • return the box with boundary wall as LEFT,UP,RIGHT,DOWN

01.     return peak_hits * (right-left+1) * (up-down+1)

00. Procedure get_binding_sites(H, proteinA, proteinB, beta):
01.     A <- len(proteinA) - 20
02.     B <- len(proteinB) - 20
            ( here H is 2-D-landscape-array ranging from 
              H[0][0](left,lower) to H[A][B](right,upper) 
              For simplicity, we are representing H[x][y] as graphical format )
03.     peak_hits <- maximum(H)    
04.     if (peak_hits == 0):
05.         return (-1, -1) (-1, -1)
                (meaning sites could not be found, 
                because previous attempts (site-1 or site-2) have exhausted the entire landscape)
06.     (x,y) <- get_indices_of_peak_of_H
07.     up, down <- y
08.     right, left <- x
09.     curr_area <- peak_hits
10.     max_area  <- get_max_area(peak_hits, up, down, left, right)
11.     while ( (curr_area > beta*max_area) && 
                (up    < B) && 
                (down  > 0) &&
                (right < A) &&
                (left  > 0)
              ):
12.         upper_bar <- get_upper_bar(H, up, down, left, right)
13.         lower_bar <- get_lower_bar(H, up, down, left, right)
14.         right_bar <- get_right_bar(H, up, down, left, right)
15.         left_bar  <- get_left_bar(H, up, down, left, right)
16.         maximum_bar <- max(upper_bar, lower_bar, left_bar, right_bar)
17.         if   (upper_bar == maximum_bar):
18.             up++ 
19.         elif (lower_bar == maximum_bar):
20.             low-- 
21.         elif (right_bar == maximum_bar):
22.             right++
23.         else
24.             left--             
25.         curr_area <- curr_area + maximum_bar
26.         max_area  <- get_max_area(peak_hits, up, down, left, right)
27.     for (x <- left to right):
28.         for (x <- down to up):
29.             H[x][y] <- 0
                    (cleaning H values so that next time this site is excluded from calculations)
30.     return (left, right) (down, up)
  • Find out the maximum point (lets call it THE hill)
  • treat this point as a unit rectangle (square in this case)
  • for this rectangle, calculate the following
    • upper bar
      • the next ‘strip’ on the upper side of this rectangle
    • lower bar
      • the next ‘strip’ on the lower side of this rectangle
    • left bar
      • the next ‘strip’ on the left side of this rectangle
    • right bar
      • the next ‘strip’ on the right side of this rectangle
  • pick the maximum bar (out of these four) and add this bar in the current rectangle
  • stop if the size/area of this rectangle is below threshold (x% of maximum possible size/area)
  • return this rectangle

helper functions

def find_top_3_connected_patches(Img, connection_size):
    01. create graph from pixels of Img where each pixel can be connected to 
            a) maximum four neighbouring pixels (up, down, left, right) and
            b) connection is only valid if the value of the neighbour is '1'
    02. this creates isolated 'islands' of connected graphs for the given Img
    03. reverse-sort these 'islands' by their size
            (size here means total number of connected pixels/nodes)
    04. filter and remove any 'island' < 'connection_size'
    05. return bounding_rectangle (defined below) for top 3 'islands'

def bounding_rectangle(points):
    01. finds minimum enclosing rectangle for given set of 2-d points
    02. let 
            a) (x,y) is coordiante of lower,left corner of this rectangle
            b) w is width of this rectangle
            c)h is height of this rectangle
    03. return (x, x+w) (y, y+h)

main function

00. def get_bright_spots(H, proteinA, proteinB, gamma):
02.     A <- len(proteinA) - 20
03.     B <- len(proteinB) - 20
04.     H <- H/maximum(H)
05.         ( normalizing the image )

06.     blurr_conv_filter_size <- (A/2, B/2)
07.     Blr <- cv2.GaussianBlur(H, blurr_conv_filter_size)
    
08.     t <- np.percentile(Blr, gamma)
09.     threshold_conv_filter_size <- (1,1)
10.     Thr <- cv2.threshold(Blr, t, threshold_conv_filter_size)
    
11.     connection_size <- min(A,B)/2
12.     return find_top_3_connected_patches(Thr, connection_size)
  • Transformation: Turn the image to grayscale (by normalize it to have values ranging between 0 to 1)
  • Convolution-Layer: apply the gaussian convolution (to blur the image)
  • Apply-Threshold: set the pixel value to 1 iff it crosses threshold or else set 0
  • *Pooling: Find the top 3 largest connected components (referred as ‘islands’ in the above algorithm detail)
    • a pixel is considered connected to another pixel if 
      • both have values = 1
      • you can reach from one to another by a single movement of either UP or DPWN or LRFT or RIGHT
  • Minimum-Enclosing-Rectangle: Return the minimum closing rectangle (of the above 3 component)

8. Another Performance metrics : What exactly is JI (or JD)

Definition

Jaccard Index (Intersection-over-union) is the area of overlap between the predicted site and the ground truth divided by the area of union between the predicted site and the ground truth.
JI = Intersection / Union
Jaccard Index is used to represents how close/similar are two segments. Since the other measure we are using (DM) is giving us the distance (or dis-similarity) between two segments, we introduced the term Jaccard Distance (JD). 
JD = 1 - JI
By doing this we can compare JD to DM since both now represent dissimilarity between predicted-binding-sites and lab-found-binding-sites

8.1. Why did we pick DM over JI (or JD)

  • As the prediction site moves further away to actual site, JI value suddenly drops to ZERO (or JD becomes ONE) beyond the point where there is no intersection. 
  • However, DM value gradually decreases as the predication site moves away from the actual interaction site (lab-site). 
  • Hence JI (or JD) does not help in capturing how close are the predictions specially in cases when there is no intersection at all, making it a very strict measuring criteria than DM

9. Tests for statistical significance of differences

9.1. Paired Sample T-Test

The paired sample t-test is also called dependent sample t-test. It’s a test to find out if there is a significant difference between 2 related variables or not. An example of this is if we want to collect the blood pressure for an individual before and after some treatment. In our case, it is whether to check if DM values are significantly different from one algorithms to another algorithms or not.

The hypothesis being test is:
  • Null hypothesis (H0): ud = 0, which translates to the mean difference between sample 1 and sample 2 is equal to 0. 
    • In our case, mean difference between DM values of algorithm-1 and algorithm-2 is equal to 0. 
  • Alternative hypothesis (HA): ud ≠ 0, which translates to the mean difference between sample 1 and sample 2 is not equal to 0.
    • In our case, mean difference between DM values of algorithm-1 and algorithm-2 is not equal to 0. 

If the p-value is sufficiently small, most commonly 0.05, one can reject the null hypothesis. In our case that means there is a significant difference between DM values of algorithm-1 and algorithm-2.

  • Here, t-value represents the difference between the two algorithms, specifically algorithm_1 - algorithm_2 , hence :
    • if t-value is negative then algorithm-1 has significant lower mean (meaning lower mean DM value) which suggests that it performs better
    • higher the t-value means more significant the difference is.

PAIRED SAMPLE T-TEST ASSUMPTIONS

In order for the paired sample t-test results to be trusted, the following assumptions need to be met:
  • The dependent variable (DV) must be continuous which is measured on an interval or ratio scale
  • The observations are independent
  • The DV should be approximately normally distributed
    • The paired sample t-test is robust to this violation. If there is a violation of normality, as long as it’s not in a major violation the test results can be considered valid
If any of these assumptions are grossly violated, a different test should be used. An alternative to the paired sample t-test is the Wilcoxon signed-rank Test.

9.2. Wilcoxon Signed-Rank Test : If we don’t assume the data fits normal distribution

The Wilcoxon signed-rank test is an alternative to the dependent t-test. It also is called the Wilcoxon T test, most commonly so when the statistic value is reported as a T value. Which scipy.stats.wilcoxon() uses for it’s calculation. This is the recommended test to use when the data violates the assumption of normality. It’s used to test if there is a significant difference on scores when there is a “before” and “after” condition of some treatment or intervention. An example of this is if we want to collect the blood pressure for an individual before and after some treatment. In our case, it is whether to check if DM values are significantly different from one algorithms to another algorithms or not.

The hypothesis being test is:
  • Null hypothesis (H0): The difference between the pairs follows a symmetric distribution.
    • In our case, DM values of algorithm-1 and algorithm-2 follows a symmetric distribution.
  • Alternative hypothesis (HA): The difference between the pairs does not follow a symmetric distribution.
    • In our case, DM values of algorithm-1 and algorithm-2 does not follow a symmetric distribution.

If the p-value is less than what is tested at, most commonly 0.05, one can reject the null hypothesis.

WILCOXON SIGNED-RANK TEST ASSUMPTIONS

In order for the Wilcoxon signed-rank test results to be trusted, the following assumptions need to be met:
  • The dependent variable (DV) must be continuous which is measured on an ordinal or continuous scale
  • The paired observations are randomly and independently drawn
  • The paired observations come from the same population

10. Insights for Walk Vs Panoramic view and Walk Vs Bright-Spot Detection

  • Walk algo works really good when there is a steep contrast in landscape-values (image)
  • Sim-Weighted normalizes the landscape-values (image) by a factor completely depending on the position/biding-site. This is not uniform for the entire image, meaning norm-factor is not a constant value for the image but for each pixel i,j its value is simProt(i)*simProt(j) This reduces the contrast of image overall reducing the performace/accuracy/TruePositiveRate of Walk-Algo
  • Since Panorama is very similar to Walk-algo. It has the same limitation (of not working well after applying SW)
  • The intuition behind using Panaroma algorithm/unwrapping algorithm/walk algorithm 2.0 etc. is when finding out the domain of protein A we take an optimal range of domain of protein B but for the original walk algorithm to find out the domain of protein A we consider only one value into account for protein B.
